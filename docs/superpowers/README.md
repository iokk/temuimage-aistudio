## Usage

1. 先看 `specs/`，确认当前产品边界和阶段结论。
2. 再看 `development/`，按维护规范修改代码和文档。
3. 做模板系统相关变更时，优先同步更新设计文档与维护文档。
4. 涉及 desktop/server 分流时，先看双运行模式设计，再修改文件能力边界。
