# UI Componentization and Action Clarity Design

## Goal

Make configuration entry clearer, reduce repeated UI logic, and explain disabled actions immediately so the product feels operable rather than mysterious.

## Scope

- Componentize `我的凭据 / 系统配置` into reusable UI sections.
- Add explicit reason prompts whenever a primary action is disabled.
- Continue the professional back-office visual refresh with more unified brand/icon treatment.

## Constraints

- Keep existing functionality intact.
- Prioritize clarity over novelty.
- Preserve admin-tool mode as the default when no database exists.
