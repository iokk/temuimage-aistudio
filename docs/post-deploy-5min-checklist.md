# 5 分钟极速验收版

适合每次 Zeabur / 服务器重新部署后，快速确认系统是不是“能用”。

## 1. 看健康状态

```bash
curl http://<your-domain-or-host>/_stcore/health
```

期望：

```text
ok
```

## 2. 看登录页

确认这 3 件事：

- 页面能打开
- 顶部显示最近更新日期和 4 个核心能力摘要
- 无数据库时应显示：`当前为管理员模式`

## 3. 管理员登录

确认：

- 管理员密码可登录
- 能进入后台
- 后台能打开 `API Key管理`

## 4. 看系统中转站配置

在后台 `API Key管理` 中确认：

- `API 地址` 还在
- `默认图片模型` 还在
- `分析/标题模型` 还在
- Key 没丢

推荐当前值：

```text
API 地址: https://newapi.aisonnet.org/v1
默认图片模型: seedream-5.0
分析/标题模型: gemini-3.1-flash-image-preview
```

## 5. 批量出图最小验证

- 上传 1 张图
- `AI分析商品` 按钮可点
- 选 1 个类型
- `AI生成图需文案` 可点

## 6. 快速出图最小验证

- 上传 1 张图
- 填商品名称
- 选 1 个类型
- `开始生成` 可点

## 7. 标题优化最小验证

- 上传图片或填商品信息
- `生成中英双语标题` 可点
- 出现标题摘要卡和标题内容

## 8. 图片翻译最小验证

- 上传 1 张图
- 当前 relay 模型如果支持翻译出图，应可直接处理
- 如果不支持，页面必须明确提示，不允许假装在跑

推荐当前 relay 图片翻译模型：

```text
seedream-5.0
```

## 9. 按钮原因提示

随便点开 `1/2/3/4` 中一个当前还不能执行的按钮，确认页面会直接显示原因：

- 没上传图片
- 没填商品名
- 没选类型
- 当前模型不支持该任务

## 10. 如果只有 1 分钟

至少确认这 5 项：

1. 健康检查 `ok`
2. 管理员能登录
3. 后台能打开
4. `1 批量出图` 能点分析
5. `4 图片翻译` 能处理，或明确提示模型不支持
